# Aya Shield 48-Hour Build Plan

## Submission Goal

Ship a believable MVP story, a visual UI flow, a concept brief, and a demo sequence that proves the idea can work.

## Must-Haves

- One-page brief
- Source list
- UI flowboard
- Demo script
- At least one lightweight prototype surface, even if seeded with example data

## Suggested Technical Slice

- Simple web page or notebook-backed demo
- Screenshot upload or sample image input
- OCR extraction step
- Rule-based or lightweight classifier for risk labels
- Result card with explanation and safe reply suggestions

## If Time Runs Out

- Keep the front end static
- Hardcode sample results
- Focus the submission on clarity of problem, usefulness, and next-step feasibility
